EXTRACTION PROCESS WORKFLOW
================================================================================

INPUT: Image-based PDF file (512 KB)
  └─> requires OCR conversion

STEP 1: OCR TEXT EXTRACTION
  - Use online service (https://ocr.space/)
  - Or local: Tesseract OCR
  - Output: extracted_text.txt

STEP 2: QUESTION PARSING
  - Run: python3 extract_luth_questions.py --input extracted_text.txt
  - Auto-detects subjects
  - Estimates difficulty
  - Output: luth_questions.json

STEP 3: VALIDATION
  - Verify all questions meet quality standards
  - Check for duplicates
  - Validate formatting
  - Review by subject

STEP 4: SQL GENERATION
  - Run: python3 extract_luth_questions.py --input luth_questions.json --output migration.sql
  - Creates database migration file
  - Output: luth_questions.sql

STEP 5: DATABASE DEPLOYMENT
  - Copy to: supabase/migrations/20260215_add_luth_exam_questions.sql
  - Deploy: supabase db push
  - Verify in Supabase dashboard

STEP 6: APPLICATION INTEGRATION
  - Test in UI
  - Verify scoring
  - Check reporting
  - Document issues

QUICK START COMMANDS

   - Upload LUTH_COMPLETE_PAST_QUESTIONS.pdf
   - Download extracted text
   - Save as extracted_text.txt

2. Parse questions:

3. View summary:
   cat luth.json | jq '.metadata'

4. Generate SQL:
   python3 extract_luth_questions.py --input luth.json --output migration.sql

5. Deploy to database:
   cp migration.sql supabase/migrations/20260215_add_luth_exam_questions.sql
   supabase db push

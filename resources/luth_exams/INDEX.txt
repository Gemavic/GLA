================================================================================
QUESTION FORMAT
================================================================================

Each question contains:
- Year: 2020, 2021, or 2022
- Subject: One of the 5 subjects above
- Question Number: Sequential within each year/section
- Question Text: Complete question (up to 500 characters)
- Options: Array of exactly 5 multiple choice options (A, B, C, D, E)
- Correct Answer: NULL (not included in extraction)
- Difficulty: 'easy', 'medium', or 'hard'
- Topic: Format like "LUTH 2022"
- Question Type: Always "multiple_choice"

Example (JSON):
{
  "year": 2022,
  "subject": "english",
  "q_num": 1,
  "text": "Without ………….words he accused him directly of treachery.",
  "options": [
    "amending",
    "modifying",
    "mixing",
    "mincing",
    "minimizing"
  ]
}

Example (SQL):
(
    1,
    'Without ………….words he accused him directly of treachery.',
    'multiple_choice',
    '["amending", "modifying", "mixing", "mincing", "minimizing"]',
    NULL,
    'medium',
    'LUTH 2022',
    2022
)

================================================================================
DATABASE TABLE STRUCTURE
================================================================================

Expected column names in Supabase:
- subject_id (INTEGER)
- question_text (TEXT)
- question_type (VARCHAR/TEXT)
- options (JSONB/JSON)
- correct_answer (VARCHAR/TEXT, can be NULL)
- difficulty (VARCHAR/TEXT)
- topic (VARCHAR/TEXT)
- year (INTEGER)

Example table creation:
CREATE TABLE exam_questions (
  id BIGSERIAL PRIMARY KEY,
  subject_id INTEGER NOT NULL,
  question_text TEXT NOT NULL,
  question_type VARCHAR(50) DEFAULT 'multiple_choice',
  options JSONB NOT NULL,
  correct_answer VARCHAR(1),
  difficulty VARCHAR(20),
  topic VARCHAR(100),
  year INTEGER,
  created_at TIMESTAMP DEFAULT NOW()
);

================================================================================
TROUBLESHOOTING
================================================================================

Q: SQL import returns encoding error
A: Ensure your SQL client is set to UTF-8 encoding

Q: Questions not importing
A: Verify table columns match: subject_id, question_text, question_type, 
   options, correct_answer, difficulty, topic, year

Q: Some special characters show as ???
A: Verify database encoding is UTF-8

Q: Total count is not 108
A: Check for duplicate imports or failed transactions

Q: Options field has errors
A: The options field must be JSONB/JSON type, not VARCHAR
